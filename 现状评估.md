# 学生论文助手 — 功能现状评估（2026-07-19）

## ✅ 已实现并正常运行

### 用户体系
- [x] 邮箱注册/登录（JWT Token，EmailStr 格式校验）
- [x] 自动注册赠送 1000 点
- [x] 全局 401 拦截（自动跳转登录页）
- [x] localStorage token 持久化

### 资产体系
- [x] 点数余额（registration 赠送 + 消费 + 充值）
- [x] 完整交易记录（CreditTransaction 模型）
- [x] 4 档充值套餐（¥10/¥50/¥100/¥300，多充多送）
- [x] 3 档订阅计划（Free/Pro ¥29/Premium ¥79）
- [x] 月付/年付切换（年付省 43%~47%）
- [x] 充值页面 UI（点数+订阅+交易记录+即将开放功能）

### 三大核心功能（完整可用）

| 功能 | 前端 | 后端 API | 亮点 |
|------|------|---------|------|
| **降重/降AIGC** | `AigcPage.tsx` | `POST /api/core/aigc` | 3种改写目标、6种检测平台、3种模型、加急 |
| **投稿前审查** | `ReviewPage.tsx` | `POST /api/core/review` | 6种期刊/会议、会议/期刊切换、审稿→修改跳转 |
| **论文修改** | `RevisionPage.tsx` | `POST /api/core/revision` | 3种修改风格（最小/标准/深度）、支持从审稿页带参数进入 |

**每个功能都有的通用能力**：
- 使用场景提示（告诉用户什么时候用）
- 余额显示 + 不足时跳充值
- 两步流程（先估算费用 → 确认后执行）
- 模型选择（DeepSeek省钱 / GPT-4o-mini标准 / GPT-4o高级）
- 加急处理选项（2 倍点数）
- 结果导出（MD/DOCX/PDF）

## ❌ 未实现

### 需后端开发的功能

| 功能 | 原因 | 实现难度 | 优先级 |
|------|------|---------|-------|
| **真实支付网关** | 当前是模拟充值，点充值按钮直接到账 | 中（支付宝当面付 0 元开通） | 🔴 高 |
| **辅助功能**（答辩模拟、格式预检、改后复查、文献综述、中译英） | 只有 UI 占位"即将开放"，无后端 API | 中（已有 `call_llm_with_config` 可复用） | 🟡 中 |
| **套餐过期自动降级** | `subscription_expires_at` 字段存在但无过期检测逻辑 | 低（每次请求时 check 即可） | 🟡 中 |
| **论文搜索/分页** | `Dashboard` 只显示前 5 篇，无搜索 | 低 | 🟢 低 |
| **使用统计/历史** | 每个功能的执行历史没有查看入口 | 低（交易记录已有，缺分类视图） | 🟢 低 |
| **消费周报邮件** | 无邮件发送能力 | 中（需 SMTP 配置） | 🟢 低 |
