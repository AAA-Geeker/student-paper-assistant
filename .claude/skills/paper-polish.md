# 论文润色 Skill

## 目标
对论文进行学术风格润色，确保术语一致、逻辑流畅、表达地道。

## 前置条件
- 论文内容已完成（在 WorkBuddy 中）
- 已读取 [[paper-style-guide]]、[[field-conventions]]

## 执行流程

### Step 1：全局风格扫描
在开始逐段润色前，先提取全文的全局特征：
- 术语清单及使用频率
- 句式特征（平均句长、主动/被动比例）
- 连接词使用情况
- 引用格式是否一致

这一步帮助后续逐段润色时保持一致性。

### Step 2：逐段润色
对每个段落进行润色，遵循 [[paper-style-guide]]：

**中文润色重点**：
- 消除口语化表达
- 统一术语翻译
- 调整句式结构（避免过长的句子）
- 增加段落之间的过渡句

**英文润色重点**：
- 时态一致性
- 主动/被动语态的合理使用
- 冠词和介词的正确使用
- 学术表达的多样性（避免重复使用同一动词）

**逐段策略**：
- 每次只传当前段落（~200-500 tokens）
- 传入前一段的最后一句（衔接用）
- 传入全文术语清单（一致性用）
- 输出润色版 + 改动标注

### Step 3：全局一致性检查
所有段落润色完成后，做一次全局检查：
- 术语是否在全文保持一致？
- 缩写是否在首次出现时展开？
- 图表编号是否连续且正确引用？
- 引用格式是否统一？

### Step 4：生成对照
可选：生成"修改前后对照"文档，标注每个改动的位置和原因。

## 输出到 WorkBuddy
```
PUT /api/papers/{id}  →  {"content": "<润色后的完整内容>"}
```

## 模型使用策略
- Step 1（扫描）：本地 NLTK/正则 分析即可（不消耗 LLM token）
- Step 2（逐段润色）：GPT-4o-mini（性价比最优）
- Step 3（一致性检查）：GPT-4o-mini，传全文术语清单
