# 论文大纲生成 Skill

## 目标
为论文生成一份详细、有逻辑链的多级大纲，确保后续写作不跑偏。

## 前置条件
- 用户已确定论文大致方向/题目
- 已读取 [[researcher-profile]]、[[advisor-feedback]]、[[field-conventions]]

## 执行流程

### Step 1：了解研究背景
读取 memory 中的研究者画像和当前进度，确认：
- 用户的论文处于什么阶段？
- 目标期刊/会议要求是什么？
- 导师的风格偏好是什么？

### Step 2：头脑风暴可能方向
如果用户还没确定具体题目，生成 3-5 个可写方向，每个方向包含：
- 核心研究问题
- 创新点概要（1-2 句）
- 可行性评估（数据、算力、时间）
- 与导师偏好的匹配度

让用户选择 1 个方向。

### Step 3：生成详细大纲
基于选定方向，生成 3 级标题的大纲：

```
1 引言
  1.1 研究背景
  1.2 研究问题
  1.3 本文贡献
2 相关工作
  2.1 子领域 A
  2.2 子领域 B
  2.3 与本文的关系
3 方法
  3.1 问题定义
  3.2 模型架构
  3.3 训练策略
4 实验
  4.1 实验设置
  4.2 主实验结果
  4.3 消融实验
  4.4 案例分析
5 结论
```

### Step 4：质量审查
对照以下清单检查大纲：
- [ ] 逻辑链是否完整？（从问题→方法→验证→结论）
- [ ] 每节是否有明确的写作目标？
- [ ] 贡献是否清晰可辨识？
- [ ] 是否避免了导师常批评的问题？（见 [[advisor-feedback]]）
- [ ] 结构是否符合目标期刊要求？（见 [[field-conventions]]）

### Step 5：输出
将大纲写入 WorkBuddy 数据库（调用 API），同时生成每节的"写作要点"提示。

## 输出到 WorkBuddy
生成完成后，通过后端 API 将大纲写入对应论文的 `outline` 字段：
```
PUT /api/papers/{id}  →  {"outline": "<大纲 Markdown>"}
```

## 模型使用策略
- Step 2-3（生成）：使用 DeepSeek（便宜、速度快）
- Step 4（审查）：使用 GPT-4o（需要质量判断能力）
