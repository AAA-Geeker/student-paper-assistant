# 学生论文助手商业化方案

## 一、痛点场景 → 收费模式

### 核心收费场景（已上线）

| 痛点场景 | 收费模式 | 定价 | 触发时机 | 月消费估算 |
|---------|---------|------|---------|-----------|
| **降重/降AIGC** — 查重标红或AI检测亮红灯 | 按字数付费 | 基础 150 点 + 80 点/千字（超出500字部分） | 每次执行 | 重度用户 2000-8000 点/月 |
| **投稿前审查** — 投稿心里没底 | 按报告付费 | 基础 500 点 + 50 点/千字（超出500字部分） | 每次生成 | 投稿季 1500-6000 点/月 |
| **论文修改** — 导师/审稿人给了意见不知怎么改 | 按方案付费 | 基础 300 点 + 40 点/千字（超出500字部分） | 每次修改 | 每月 1000-4000 点/月 |
| **加急处理** — 最后一刻赶 deadline | 2倍点数 | 以上单价 × 2 | 用户勾选 | 按需 |

### 辅助功能场景（待开发）

| 痛点场景 | 建议收费模式 | 建议定价 | 优先级 |
|---------|-------------|---------|-------|
| **投稿格式预检** — 被退稿说格式不符 | 按次 ¥5-10（500-1000点） | 固定收费 | 高 |
| **改后复查** — 改完了不知道够不够 | 按次 ¥3-8（300-800点） | 固定收费 | 高 |
| **答辩模拟** — 不知道答辩会问什么 | 按次 ¥10-20（1000-2000点） | 固定收费 | 中 |
| **文献综述生成** — 没时间读文献 | 按摘要数量 ¥5-15/篇 | 按篇收费 | 中 |
| **中译英学术润色** — 英文论文表达不地道 | 按字数 ¥8-20（800-2000点） | 按字数收费 | 中 |
| **论文大纲生成** — 不知道论文结构怎么搭 | 按次 ¥3-5（300-500点） | 固定收费 | 低（已有免费工具替代） |

## 二、商业化体系结构

### 2.1 点数经济（虚拟货币）

```
1 元人民币 = 100 点
注册赠送 = 1000 点（≈10 元体验额度）
加急费   = 正常点数 × 2
```

### 2.2 充值套餐（4 档）

| 套餐 | 价格 | 到手点数 | 赠送 | 性价比 |
|------|------|---------|------|-------|
| 体验包 | ¥10 | 1000 点 | 0 | 100 点/元 |
| 标准包 | ¥50 | 6000 点 | 500 点赠 | 120 点/元 |
| 专业包 ⭐推荐 | ¥100 | 14000 点 | 2000 点赠 | 140 点/元 |
| 团队包 | ¥300 | 42000 点 | 7000 点赠 | 140 点/元 |

### 2.3 订阅计划（3 档）

| 计划 | 月付 | 年付 | 每日免费核心功能 | 辅助功能折扣 | 核心价值主张 |
|------|------|------|----------------|-------------|------------|
| 免费版 | ¥0 | — | 0 次 | 无折扣 | 按次付费体验 |
| Pro ⭐ | ¥29 | ¥199（省43%） | 3 次 | 8 折 | **每天 3 次核心功能免费，适合每周写论文的本科生/硕士** |
| Premium | ¥79 | ¥499（省47%） | 10 次 | 8 折 | **每天 10 次核心功能免费+优先响应，适合密集写作的博士/科研人员** |

### 2.4 核心营收公式

```
单用户月营收 = 充值收入 + 订阅收入
充值收入 = (总消费点数 - 订阅赠送免费次数) × 点数单价
订阅收入 = 月/年订阅费

毛利率 = (营收 - LLM API 成本) / 营收
```

**典型用户画像月营收：**

| 用户类型 | 月使用场景 | 月消费点数 | 最佳方案 | 月营收 |
|---------|-----------|-----------|---------|-------|
| 轻度用户（本科） | 每月 2 次降重 + 1 次审查 | ~1500 点 | 标准包 ¥50/月 | ¥50 |
| 中度用户（硕士） | 每周 3 次降重 + 2 次修改 | ~6000 点 | Pro 年付 ¥16.6/月 + 标准包 ¥50/季度 | ~¥33/月 |
| 重度用户（博士） | 每天使用（降重+审查+修改） | ~15000 点 | Premium 月付 ¥79 | ¥79/月 |
| 团队（课题组） | 多人共用 | ~40000 点 | 团队包 ¥300/月 | ¥300/月 |

## 三、付费转化路径

```
免费体验（注册送 1000 点）
  ↓ 用完体验点数
余额不足提示 → 充值页面
  ↓ 高频用户
订阅引导 → 对比免费/Pro/Premium
  ↓ 紧急需求
加急选项（日常消费 2 倍）
```

### 付费触发点设计

1. **余额不足弹窗** — 三大核心页面都有余额检查和充值引导（已实现）
2. **预估后引导** — 每次使用前先估算费用，让用户确认后再扣费（已实现）
3. **订阅对比页** — 清晰展示 3 档计划的权益差异（已实现）
4. **加急选项** — 所有核心功能底部都有加急勾选框（已实现）

### 待实现的付费触发点

5. **首次使用引导弹窗** — 用户第一次使用某个功能时，弹出"这个功能花多少点"的说明
6. **消费周报** — 每周邮件通知用户本周消费了多少点、用在什么场景
7. **余额不足预约** — 余额不足时让用户预约，到账后自动执行
8. **年付早鸟价** — 开学季/论文季的年付折扣活动

## 四、支付网关接入建议

### 最低成本方案：支付宝当面付

| 项目 | 说明 |
|------|------|
| 接口 | 支付宝当面付（扫码支付） |
| 费用 | 0 元开通，0.6% 手续费 |
| 接入难度 | 低，有官方 SDK |
| 适用场景 | PC 端扫码、移动端跳转 |

### 推荐集成方案

```
用户选择套餐 → 生成订单 → 返回二维码 → 用户扫码支付
  → 支付宝异步回调 → 验证签名 → 更新用户点数/订阅
  → 前端轮询订单状态 → 显示充值成功
```

后端已预留的接口：
- `POST /me/top-up` — 接收 `package_id`，当前直接到账
- `POST /me/subscribe` — 接收 `plan`，当前直接激活
- 只需在 `commerce.py` 的 `apply_top_up` 和 `apply_subscribe` 中插入支付验证即可

## 五、风险与限制

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| API 成本波动（DeepSeek 涨价） | 毛利率下降 | `model_router.py` 已有多模型降级逻辑 |
| 用户刷免费次数 | 收入流失 | 每日免费次数有 DB 记录上限 |
| 支付网关接入延迟 | 无法真实营收 | 先用模拟充值验证商业模式 |
| 替代品（知网AI、ChatGPT 降重）竞争 | 用户流失 | 专注"先估后执行+场景化引导"的体验差异 |
